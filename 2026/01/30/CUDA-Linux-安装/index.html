CUDA - Linux 安装

发表于 2026-01-30 分类于 工作环境配置 ， CUDA
Linux 安装 CUDA。

Linux 上 CUDA 的安装

参考链接：Linux服务器非root用户下安装CUDA11.1和cudnn到指定目录_cudnn目录_BUAA～冬之恋的博客-CSDN博客。

  • 查看机器型号确定安装版本

    uname -a 
lsb_release -a

  • 创建下载目录：

    mkdir ~/cuda-11.6
mkdir ~/cuda-11.6/my_lib
mkdir ~/cuda-11.6/my_install

  • 根据Linux系统的版本，去CUDA官网选择下载对应的的 CUDA：

    假设我们期望安装 11.6 版本的 CUDA，则我们选择 Linux - x86_64 - Ubuntu - 18.04 - runfile(local) 进行下载，并将下载好的 .run 文件保存在 ~/cuda-11.6/my_install/ 路径中，然后输入如下命令：

    cd ~/cuda-11.6/my_install
bash cuda_11.6.2_510.47.03_linux.run

  • 等待一段时间，会有安装提示窗口出现，输入 accept。注意，此安装提示窗口会比较大，最好最大化终端窗口，使其占据全屏。

  • 选择 accept 后，更改安装选项：

    X 代表选中，第一项是显卡驱动，这里我只安装 CUDA Toolkit和 CUDA Samples，一般不搞 CUDA 编程，仅用来跑深度学习就够了

    CUDA Installer                                                          
 - [ ] Driver                                                            
      [ ] 455.32.00                                                       
 + [X] CUDA Toolkit 11.*                                                 
   [X] CUDA Samples 11.*                                                
   [ ] CUDA Demo Suite 11.*                                               
   [ ] CUDA Documentation 11.*                                          
   Options                                                               
   Install
### 这一步中，不同版本的CUDA可能界面会有所不同，请谨慎甄别.
    • 选中 Options
      • 先更改 Toolkit Options ，将 /usr 这种目录的去掉（全部去掉）
      • 另外进入 Change Toolkit Install Path 设置cuda安装到自己具有写入权限的路径，我这里是 /data/huzihao/cuda-11.6/（这里最好使用绝对路径）
    Change Toolkit Install Path
/data/huzihao/cuda-11.6/

Library install path (Blank for system default)
/data/huzihao/cuda-11.6/my_lib

  • 安装完毕后，添加环境变量 vim ~/.bashrc

    export PATH="/data/huzihao/cuda-11.6/bin:$PATH"
export LD_LIBRARY_PATH="/data/huzihao/cuda-11.6/lib64:/data/huzihao/cuda-11.6/my_lib/lib64:$LD_LIBRARY_PATH"

    然后 source ~/.bashrc

  • 测试 CUDA：nvcc -V

Linux 上 cuDNN 的安装

参考链接：linux 安装 CUDNN_linux安装cudnn-CSDN博客

官网下载相应版本，要与 CUDA 版本兼容：cuDNN Archive | NVIDIA Developer

假设我已经安装好 cuda-11.6，现在我希望安装 cuDNN v8.5.0 (August 8th, 2022), for CUDA 11.x，在官网中选择这一栏，再选择 Local Installer for Linux x86_64 (Tar) 下载到本地即可

  • 下载路径：~/cudnn/

  • 进行安装

    cd ~/cudnn
tar -xf cudnn-linux-x86_64-8.5.0.96_cuda11-archive.tar.xz # 需要一定时间
cd cudnn-linux-x86_64-8.5.0.96_cuda11-archive

cp ./include/* ../cuda-11.6/include
cp ./lib/libcudnn* ../cuda-11.6/lib64

chmod a+r ../cuda-11.6/include/cudnn.h ../cuda-11.6/lib64/libcudnn*

注意

安装结束后，我们会发现较旧版本的 cuDNN 目录与较新版本的 cuDNN 目录不一致

旧版本：

cuda
----include
---------cudnn.h
----lib64
--------...

新版本：

cuda
----include
---------cudnn.h
---------cudnn_*.h
----lib64
--------...

修改 cuDNN 为新安装的版本

  • 找到对应的 cuDNN 文件夹

  • 将 cuDNN 的文件复制到 CUDA 文件夹中：

    sudo cp ./include/* /usr/local/cuda-10.2/include
sudo cp ./lib/libcudnn* /usr/local/cuda-10.2/lib64
sudo chmod a+r /usr/local/cuda-10.2/include/cudnn.h /usr/local/cuda-10.2/lib64/libcudnn*
    • 注意：低版本的 cuDNN 和高版本的 cuDNN 的文件结构不完全相同，需要仔细甄别

  • 需要执行一下 sudo ldconfig，以更新依赖。

    • 若出现错误如下：

      /sbin/ldconfig.real: /data/lyq/cuda-10.2/targets/x86_64-linux/lib/libcudnn.so.7 is not a symbolic link

      则进行如下操作：

      cd /data/lyq/cuda-10.2/lib64
sudo rm libcudnn.so
sudo rm libcudnn.so.7
sudo ln libcudnn.so.7.6.5 libcudnn.so.7
sudo ln libcudnn.so.7 libcudnn.so
sudo ldconfig # No news is good news. :)

      这里的libcudnn.so.7.6.5不一定是7.6.5版本的，在操作前请检查.../lib64/下对应libcudnn的具体版本：通过find . -name "*cudnn*"可检查。

    • 若没有这个CUDA对应的.config，则cd /etc/ld.so.conf.d，按如下方式进行创建：

      假设CUDA目录为/data/usr/cuda-10.2

      sudo touch cuda-10-2.conf
sudo vim cuda-10-2.conf
# 在上述文件中输入/data/usr/cuda-10.2/targets/x86_64-linux/lib
# 保存、退出即可
